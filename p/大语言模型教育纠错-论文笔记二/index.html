2305.12962 Distilling ChatGPT for Explainable Automated Student Answer Assessment

  • argu使用bert模型分类器对学生打分不好（看不见过程、可信度低）
  • 现存数据集只有分数标注、无cot标注

paper贡献

  • AERA 框架，ChatGPT蒸馏到更小的模型
  • 两种ChatGPT自主优化推理的策略
  • 实验与人工评估，无需外部标注数据，即可生成高质量评分依据

To the best of our knowledge, AERA is the pioneering framework that leverages ChatGPT to generate rationales for explainable student answer assessments using more compact language models.

我们的方法包含三个主要步骤：

  • 根据不同的推理难度级别设计多种提示模板，指导 ChatGPT 在评估学生答案时同步生成评分依据
  • ChatGPT 可能无法准确评估所有学生答案。引入了依据与数据优化模块，旨在提升生成依据的质量和可用性
  • 用于高效微调小型语言模型，从而为学生答案评估生成合理的解释依据。

Data & Rationale Refinement

  • 模型的输出每次都有不确定性，为保证评估分数的合理性，本章节着重讨论了数据与理由Refinement模块

策略1：Fixing Wrongly Labelled Data

引用了2302.09664 Semantic Uncertainty: Linguistic Invariances for Uncertainty Estimation in Natural Language Generation

采用该论文提到的LLMs语义置信区间来计算生成解释对应的评分不确定性

作者对同一个学生答案进行多次（或多次假设）评估，以观察其输出分布。

发现：分数相同的情况下，给出的理由在语义上是相似的

因此：将这些具有语义相似理由和相同预测分数的实例的概率进行累加，从而得到该分数的总预测概率。

$$ p(\hat{y}_{i}|x_{i})=\sum_{\hat{y}_{i}\in S}p(\hat{y}_{i}|x_{i}) $$

其中 $S$ 表示所有语义相似且预测分数相同的解释集合。

当ChatGPT高度自信时，打出的分数与人类标注的分数差值超过1时，认为人类错误

从而修正错误的标注数据集

  • 携带附加信息整体性能没有提升

    • 模型可能参考虚假特征而非参考评分标准
    • 仅基于学生答案进行训练时，采用能处理更长输入文本的模型可以提升性能。

  • 提示词的优化确实提升了ChatGPT性能

  • paper提出的框架得到的微调模型在部分数据集上没能超过纯分类器（只打分不解释，指标肯定更好看），但是也有部分超过

  • 如果只保留预测分数正确的理由子集（“Correct Score Only”），在某些数据集（#1、#5和#6）上性能会超过包含其他细化策略的情况

    • 这种方法在数据量有限时（如数据集#2）可能不适用
    • 消融实验证明：分别纳入“修正错误标签”（Fixing Wrong Labels）和“理由完善”（Rationale Refinement）两种策略后，数据集#2的性能由于数据可用性增加而显著提升。
      • 在数据集#1、#5和#6上观察到性能下降，这表明错误标记的数据或不准确的预测理由可能对整体性能产生负面影响

数据与理由完善的两个组件在框架中都至关重要，既能防止数据稀缺，又能有效减少噪声数据

参考阅读：

论文的消融研究旨在评估数据与理由完善模块中各个组件的贡献。这里提到的“四个微调方案”通常指的是在Long T5模型微调的基础上，逐步引入或移除特定完善策略的配置。

具体来说，这些方案及其差异体现在：

  1. “Correct Score Only” (仅保留正确分数的数据)：
    • 方案：在这种配置下，只使用ChatGPT生成的理由中，其预测分数与真实人工标注分数完全匹配的那部分数据来微调Long T5模型。这意味着任何ChatGPT给出了错误分数（即便理由可能在某种程度上正确）的数据点都被排除。
    • 目的：这是为了建立一个高置信度的基线，确保用于微调的数据本身质量很高（分数是正确的）。
    • 差异：这是最严格的数据筛选策略，数据量会相对减少。
  2. 引入“Fixing Wrong Labels” (修正错误标签)：
    • 方案：在“Correct Score Only”的基础上，引入了利用LLM语义置信区间来识别并修正人工标注中潜在错误的数据。这意味着即使ChatGPT最初给出的分数与人工标注不符，但如果ChatGPT对该评估高度自信（基于语义相似的理由），并且分数差异超过某个阈值（如1分），则认为可能是人工标注错了，并尝试修正。
    • 目的：增加可用数据的数量，将部分原本被排除的“错误”数据点（因人工标注问题）重新纳入训练集。
    • 差异：在数据筛选上比“Correct Score Only”更宽松，因为它试图“拯救”因人工错误导致的数据。
  3. 引入“Rationale Refinement” (理由完善)：
    • 方案：在“Correct Score Only”的基础上，引入了利用“理由完善提示词”来优化ChatGPT生成的理由。即便ChatGPT最初生成的理由不够完美，但通过提供正确的学生答案分数作为先验知识，引导模型生成更精确、与分数更匹配的理由。
    • 目的：提高理由本身的质量和准确性，使其更好地解释给定分数。
    • 差异：关注的是理由内容的优化，而非分数的修正或数据量的增加。
  4. AERA完整方案：
    • 方案：这是将“Fixing Wrong Labels”和“Rationale Refinement”这两个组件同时整合进来的最终框架。它既尝试修正底层数据的问题，也优化了理由的质量。
    • 目的：实现框架的整体最优性能，同时解决数据量和理由质量的问题。
    • 差异：是前述组件的协同作用。

为什么证明两个组件在框架中都至关重要（引入一个组件不是造成了性能损失吗）

用户提出的问题非常关键，因为它指出了一个看似矛盾的地方：论文指出单独引入某个组件可能导致部分数据集的性能下降，但又强调两个组件都至关重要。这其中的考量是多方面的：

  1. 数据稀缺性问题：
    • “Correct Score Only”策略虽然能确保数据质量极高，但它会大幅减少可用数据的数量。对于像数据集#2这样本身就数据量相对较小的场景，严格的“Correct Score Only”会导致模型训练数据不足，从而严重限制性能。
    • 引入“Fixing Wrong Labels”和“Rationale Refinement”的第一个关键作用在于：它们能够将那些被“Correct Score Only”策略排除在外的数据（例如，因人工标注错误导致被误判为“错误”的数据，或ChatGPT最初理由不完美但分数可能正确的数据）重新纳入训练集。这在数据稀缺的情况下，极大地增加了可用于训练的数据量。
    • 因此，尽管单独引入这些策略在某些数据充足的场景下可能看不出明显优势甚至有轻微下降，但它们是防止数据稀缺、保证框架在各种数据集（特别是小数据集）上都能有效运行的关键。
  2. 数据噪声与质量权衡：
    • 论文也确实提到，当“Fixing Wrong Labels”和“Rationale Refinement”策略单独引入时，在一些数据集（如#1、#5、#6）上观察到性能下降。这可能是因为：
      • “Fixing Wrong Labels”虽然修正了部分人工错误，但如果ChatGPT自身的判断并非100%完美，也可能引入新的噪声。
      • “Rationale Refinement”虽然提高了理由质量，但如果与原始数据的匹配度没有处理好，也可能导致一些复杂性。
    • 这种“性能损失”实际上是引入更多数据和更复杂处理可能带来的副作用，表明单独的组件可能无法完美处理所有情况。
  3. 两个组件的协同作用与整体重要性：
    • 论文强调两个组件都至关重要的原因在于它们的互补性和在整体框架中的协同作用：
      • “Fixing Wrong Labels” 主要解决的是底层数据质量和可用性问题，它使得更多的数据点可以被信赖和使用，即使它们最初的人工标注存在问题。这扩展了训练数据的池子。
      • “Rationale Refinement” 主要解决的是生成理由的准确性和一致性问题，它确保了那些被用于模型蒸馏的理由是高质量、高解释性的。
    • 一个关注数据量的扩展和纠错，另一个关注数据内容的优化。
    • 当两者结合在AERA的完整框架中时，它们能够更全面地应对真实世界数据存在的挑战——既要处理数据稀缺和潜在的标注错误，又要确保用于训练“学生”模型的理由具有高质量和可解释性。

因此，即使单独引入某个组件在特定情况下未能带来普遍的性能提升甚至略有下降，但从应对不同数据集特性（尤其是数据量大小）和确保最终模型的可解释性与可靠性的全局角度来看，这两个组件在AERA框架中都发挥着不可或缺的关键作用。它们的价值体现在提供了处理复杂数据和实现多任务（打分+理由生成）的鲁棒解决方案。

自己的想法：paper准确的发现影响性能的两个关键要素：

  • 数据集本身的错误
  • ChatGPT的错误

只解决一个问题反而会带来性能损失
